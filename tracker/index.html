The Mission
What Are We Building & Why?
A web app where any student pastes a YouTube lecture URL, types a question with a timestamp, and gets an expert-quality AI answer — for free.

📝 What the Paper Did

  • Built EduVidQA dataset (5,252 QA pairs from CS lectures)
  • Tested 6 models (GPT-4o, Gemini, mPLUG-Owl3, etc.)
  • Found: synthetic fine-tuning lets 8B models match GPT-4o
  • Created Clarity/ECT/UPT evaluation metrics
  • But: no RAG, 256 token cap, CS-only, no product

🚀 What We're Building

  • Full RAG pipeline: embed lectures → retrieve → generate
  • Qwen2.5-VL-7B (free, open-source, runs on M2 Mac)
  • Multimodal: transcript + video frames + slides
  • 4096 token answers (not 256)
  • Real web app anyone can use — $0 total cost
💡 Why this matters: 67.9% of students watching lectures WANT to ask questions but can't get answers. We're turning a research paper into a tool that solves this. If it works, any student worldwide can have a free AI TA.

📋 Request Flow (Step by Step)

  1. Student enters YouTube URL + timestamp + question on the frontend
  2. Backend checks cache — if this video was processed before, skip to step 5
  3. Pipeline/Ingest downloads transcript (youtube-transcript-api) + extracts key frames (yt-dlp + ffmpeg)
  4. Pipeline/RAG chunks transcript into 2-min segments, embeds each chunk (BGE-M3), stores in ChromaDB
  5. Pipeline/RAG embeds the student's question, retrieves top-5 most relevant chunks + their frames
  6. Inference feeds retrieved chunks + frames + question to Qwen2.5-VL-7B → generates answer (max 4096 tokens)
  7. Backend returns answer + quality scores (Clarity/ECT/UPT) to frontend
  8. Frontend displays formatted answer with source timestamps, confidence, and quality badges

Decision Log
Why We Chose What We Chose
Every technical decision with alternatives considered and reasons documented.
🧠 Which model for answer generation?
Qwen2.5-VL-7B-Instruct. It's the successor to Qwen2-VL (used in the paper), supports hour-long video understanding, dynamic FPS, and runs on M2 16GB with 4-bit quantization (~5GB VRAM).
✅ Qwen2.5-VL-7B-Instruct (4-bit)
Alternatives considered: LLaVA-OneVision 7B (good but less video support), InternVL3 (heavier), Gemini Flash (free API but rate-limited and closed). Qwen2.5-VL wins on video capability + local deployment + open weights.
🔍 Which embedding model for RAG retrieval?
BGE-M3 (568M params) — multilingual, multi-granularity, works great for educational text. Falls back to all-MiniLM-L6-v2 (22M) if memory is tight.
✅ BGE-M3 (primary) + MiniLM (fallback)
Alternatives: Gemini Embedding 2 (best quality but requires API key + costs money eventually), OpenAI ada-002 (costs money), E5-large (good but monolingual). BGE-M3 is free, local, and top-tier on MTEB.
🗄️ Where to store embeddings?
ChromaDB — lightweight, Python-native, stores in SQLite on disk. No server to run, no cloud costs. Perfect for demo-scale (thousands of chunks per lecture).
✅ ChromaDB (local SQLite)
Alternatives: Qdrant (needs Docker), Pinecone (cloud, paid), FAISS (no metadata filtering), Weaviate (heavy). ChromaDB is the simplest free option.
🖥️ Where to run GPU inference for free?
HuggingFace Spaces with ZeroGPU. Gets free A10G/T4 GPU bursts on demand. Auto-sleeps when idle (no cost). Persistent URL — doesn't disconnect like Colab.
✅ HuggingFace Spaces (ZeroGPU)
Alternatives: Kaggle (30hrs/week but can't expose API), Colab (disconnects after 90min), Local M2 (works for dev but 7B inference is slow on CPU). ZeroGPU is the only free option giving persistent GPU API.
🌐 Where to host the frontend?
Vercel free tier — auto-deploys from GitHub, free SSL, edge CDN, perfect for React/Next.js. 100GB bandwidth/month is plenty.
✅ Vercel (free tier)
Alternatives: GitHub Pages (static only, no SSR), Netlify (similar but Vercel has better React support), Cloudflare Pages (good but less familiar).
📝 How to get transcripts?
youtube-transcript-api first (instant, free, no download needed). If no captions available, fall back to yt-dlp audio download + Whisper-small transcription.
✅ youtube-transcript-api + Whisper fallback
Most educational videos (NPTEL, MIT OCW, Stanford) have captions. Whisper-small runs on M2 CPU in real-time. No API keys needed for either.
📏 How to evaluate answer quality?
Use the paper's own Clarity/ECT/UPT Likert scales. Score using a free LLM (Qwen2.5-72B via HF Inference API or Groq free tier). Show scores as badges in the UI.
✅ Paper's Likert metrics + free LLM scorer
This directly connects our product to the paper's contribution (C3). We validate our answers using the same metrics the paper invented.
