Where We Are
Status as of 25 April 2026
The research-paper pipeline is built and working end-to-end on a single machine. The next chapter is turning it into a real product anyone can sign up for.

✅ What's done

  • Full multimodal RAG pipeline (ingest → keyframes → chunking → embed → retrieve → answer → score)
  • FastAPI backend with 3 endpoints (/api/health, /api/process-video, /api/ask)
  • React + Vite + Tailwind frontend (single-screen player + chat)
  • 5 lectures already indexed in local Chroma store
  • Pytest suite covering each pipeline stage + e2e
  • Manim explainer videos v1 → v4 for supervisor demo
  • HANDOFF.md, ROADMAP.md, ARCHITECTURE.mmd documented

🚧 What's next

  • Supabase Auth (Google + magic link) — first real users
  • Postgres + pgvector — replaces local Chroma, enables global dedup
  • Supabase Storage for keyframes — works across servers
  • Background worker + queue — heavy work out of HTTP handlers
  • Landing / library / watch / review page routing
  • Checkpoint quizzes + on-demand "Test me" + spaced review
  • Deploy: Vercel (FE) + Supabase (DB/Auth/Storage) + HF Space (API+worker)
⚠️ Blockers before any public deploy: 3 leaked API keys still in .env need rotation · CORS is * · no auth on paid LLM endpoints · server errors leak to clients · vercel.json rewrite is broken. See Known Debt.

🔑 Invariant 1
A video is processed once globally. Dedup via UNIQUE (video_id, pipeline_version) + INSERT … ON CONFLICT DO NOTHING as a free distributed lock.
🔑 Invariant 2
Quiz cache key is global, not per-user: (video_id, ts_bucket_30s, prompt_version). Two users pausing at the same moment share the same generated quiz.
🔑 Invariant 3
No long work in HTTP handlers. Anything >2 s goes to pgmq + a worker. The API stays snappy under load.
🔑 Invariant 4
user_id is the partition key on every user-owned row. Today it's a single Postgres; tomorrow it shards mechanically.

Known Debt
Things to Fix Before Public Launch
Sourced from HANDOFF.md. Ordered by severity. Red = blocker · amber = MVP-needed · cyan = code hygiene.
P0
Three live API keys still in .env

HF_TOKEN, GROQ_API_KEY, GEMINI_API_KEY. .env is gitignored and not in history, but ownership changed hands — assume compromised. Rotate before any deploy.

P1
CORS = * with credentials

backend/app.py uses allow_origins=["*"] together with allow_credentials=True — browsers reject this combo and any origin can call paid LLM endpoints. Lock to the Vercel domain(s).

P1
No auth on /api/ask or /api/process-video

Both endpoints trigger paid LLM calls. Add Supabase JWT verification.

P1
Server errors leak to clients

Several HTTPException(500, detail=str(exc)) paths return file paths and provider SDK errors. Log full exception server-side, return a generic message.

P2
Backend ↔ frontend contract mismatches (silent UI breakage)

generation_time_seconds vs generation_time · ProcessResponse shape mismatch · HealthResponse.model_name missing in FE type. Fix root cause: generate TS types from OpenAPI.

P2
vercel.json rewrite is broken

Uses ${VITE_API_URL} in destination — Vercel does not expand build-time env vars in rewrites at runtime. Hardcode or delete the rewrite.

P2
_last_error bug in pipeline/answer.py

Variable referenced in final raise but only assigned in the Groq except block. Initialise _last_error = None at top of generate_answer.

P3
v1 pipeline still in repo alongside v2

Delete: pipeline/{rag,embeddings,evaluate,inference,inference_gemini,inference_gemini_video,inference_groq}.py. Rename v2 modules to canonical names. Update backend/app.py imports.

P3
process-video blocks the event loop

It's async def but does sync yt-dlp / OpenCV / SSIM / embedding / Chroma work on the request coroutine. Move to background task / queue.

P3
Unpinned requirements.txt + fragile Jina monkey-patch

Builds aren't reproducible; any HF cache refresh re-breaks the EVA model patch in embeddings_v2.py. Pin versions, fork model, or fail loudly.

P4
Dead frontend code + Dockerfile runs as root

components/VideoInput.tsx, hooks/useAskQuestion.ts, @tanstack/react-query dep all unused. Dockerfile lacks non-root USER.
